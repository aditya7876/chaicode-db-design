# 🚀 Elevator Management System - Database Design

This project presents a scalable **ER diagram & database schema** for managing elevators in multi-floor buildings.

## 📌 Overview
The system is designed to handle:
- Building & floor management
- Elevator operations & assignments
- Ride tracking & logs
- Maintenance records
- Real-time elevator status

## 🧩 Key Entities
- **Building** – Stores building details
- **Floor** – Represents floors within buildings
- **Elevator Shaft & Elevator** – Physical and operational units
- **Floor Request** – User requests (up/down)
- **Ride Assignment** – Assigns elevators to requests
- **Ride Log** – Tracks completed rides
- **Elevator Status** – Real-time state tracking
- **Maintenance Record** – Logs servicing and issues
- **Elevator Floor Coverage** – Maps elevators to floors

## ⚙️ Features
- Supports multiple buildings and elevators
- Tracks ride lifecycle (request → assignment → completion)
- Handles maintenance and downtime
- Enables real-time elevator monitoring

## 🛠️ Tech
- ER modeling using tools like **Eraser**
- Relational database design (PostgreSQL-friendly)
